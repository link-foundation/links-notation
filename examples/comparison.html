Format Comparison: Representing Family Data

The same structured data represented in different formats

LiNo • YAML • JSON • XML

LiNo (Links Notation)
family: (parents: (father: John) (mother: Mary))
family: (children: (son: Tom) (daughter: Alice))
(John loves Mary)
(Mary loves John)
(parents love children)
pets: (dog: Rex) (cat: Whiskers)
(Rex belongs_to family)
(Whiskers belongs_to family)
address: (street: "123 Main St") (city: Springfield) (state: Illinois)
YAML
family:
  parents:
    father: John
    mother: Mary
  children:
    son: Tom
    daughter: Alice

relationships:
  - subject: John
    predicate: loves
    object: Mary
  - subject: Mary
    predicate: loves
    object: John
  - subject: parents
    predicate: love
    object: children

pets:
  dog: Rex
  cat: Whiskers

pet_ownership:
  - subject: Rex
    predicate: belongs_to
    object: family
  - subject: Whiskers
    predicate: belongs_to
    object: family

address:
  street: "123 Main St"
  city: Springfield
  state: Illinois

LiNo Advantages

  • Natural: Reads like human language - "John loves Mary"
  • Concise: Most compact representation with clear relationships
  • Intuitive: Links and references are expressed naturally
  • Flexible: Handles any number of references and nested structures
  • Universal: Can represent doublets, triplets, and N-tuples seamlessly
